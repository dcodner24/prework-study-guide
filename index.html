HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • Items in the head element are not visibly to users.
  • The abbreviations ul, ol, and li can be used to make different kinds of lists and add itms to them.
  • Different heading levels can be created by using the abbreviations h1, h2, and h3.

CSS

  • CSS represents the style/special formatting on a webpage.
  • A margin indicates now much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • Images can be refined using CSS. This results in neater formatting. (img{})
  • Shadows and borders can be added to lists to add more depth by using the following formatting: border: 5px solid gray; box-shadow: 5px 10px #888888;

Git

  • git status: checks what branch we are currently on and warns the user of any changed filed that are not added to the next commit.
  • git checkout -b branch-name: Creates a new branch and switches to it. (-b denotes a new branch)
  • git pull origin "branch": Pulls the files from the specified branch in a repo to whatever branch the user is currently on.
  • git push origin "branch": Pushes the commited files from the specified local branch to GitHub.

JavaScript

  • JavaScript acts as the brains of a webpage, adding functionality.
  • If and Elif statements can be used alongside variables to perform calculations and debug code.
  • For loops can be used in conjunction with functions to streamline and automate code.
